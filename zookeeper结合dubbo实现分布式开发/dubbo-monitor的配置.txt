1、从github上克隆dubbo源码带、/tmp/dubbo目录中,并且自己完成打包构建
git clone https://github.com/alibaba/dubbo.git
cd dubbo
mvn clean install -Dmaven.test.skip

2、从dubbo/dubbo-simple/dubbo-monitor-simple/target/中将dubbo-monitor-simple-2.5.5-assembly.tar.gz拷贝到/mnt/soft中
再将jdk-8u11-linux-x64.tar.gz也拷贝到该目录下，创建Dockerfile文件，录入如下信息：




#pull down centos image
FROM centos
MAINTAINER 565599455@qq.com
#copy jdk and monitor into image
ADD ./jdk-8u11-linux-x64.tar.gz /usr/local/java
ADD ./dubbo-monitor-simple-2.5.5-assembly.tar.gz /usr/local/
#set environment variable
ENV JAVA_HOME /usr/local/java/jdk1.8.0_11
ENV JRE_HOME $JAVA_HOME/jre  
ENV CLASSPATH .:$JAVA_HOME/lib:$JRE_HOME/lib  
ENV PATH $PATH:$JAVA_HOME/bin
#define entry point which will be run first when the container starts up
ENTRYPOINT /usr/local/dubbo-monitor-simple-2.5.5/bin/start.sh



//构建镜像
docker build -t puxubo/monitor:latest --rm=true .


//启动容器
docker run -d -p 8090:8080 --name dubbo-monitor --link zookeeper_regiester:zookeeper_regiester -v /etc/soft/dubbo.properties:/usr/local/dubbo-monitor-simple-2.5.5/conf/dubbo.properties puxubo/monitor:latest



3、访问地址
http://192.168.43.100:8090/


注意：dubbo通常使用dubbo-monitor，dubbo-admin来共同管理负载，以及压力的监控
